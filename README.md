# Azure Terraform Example – Provisioning Infrastructure with Terraform

This project demonstrates how to create Azure resources using **Terraform**, a widely used Infrastructure-as-Code (IaC) tool.

Instead of manually configuring resources in the Azure Portal, this example shows how to define and deploy infrastructure in a **declarative, repeatable way**.

---

## 📌 What This Project Does

This Terraform configuration provisions the following resources in Azure:

* A **Resource Group**
* A **Storage Account** inside that Resource Group

The goal is to illustrate how Terraform can be used to automate infrastructure creation.

---

## 🧠 Key Concept: Infrastructure as Code

Terraform allows you to define infrastructure using code.

Instead of performing manual steps, you describe the desired state:

> “I want these resources with this configuration.”

Terraform then:

* compares your configuration with the current state in Azure
* determines what needs to change
* applies those changes safely

---

## 🏗️ Resources Created

### 1. Resource Group

```hcl
resource "azurerm_resource_group" "example" {
  name     = "my-terraform-rg"
  location = "France Central"
}
```

Creates a Resource Group named:

* `my-terraform-rg`
* located in `France Central`

---

### 2. Storage Account

```hcl
resource "azurerm_storage_account" "example" {
  name                     = "mysac65754273ab"
  resource_group_name      = azurerm_resource_group.example.name
  location                 = azurerm_resource_group.example.location
  account_tier             = "Standard"
  account_replication_type = "GRS"

  tags = {
    environment = "staging"
  }
}
```

Creates a Storage Account with:

* **Tier**: Standard
* **Replication**: GRS (Geo-Redundant Storage)
* **Tag**: `environment = staging`

---

## 🔗 Resource Relationships

Terraform automatically understands dependencies:

* The Storage Account depends on the Resource Group
* It references:

  * `azurerm_resource_group.example.name`
  * `azurerm_resource_group.example.location`

This ensures correct creation order without manual intervention.

## 🔌 Provider Configuration

```hcl
terraform {
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "4.64.0"
    }
  }
}
```

* Defines the Azure provider (`azurerm`)
* Locks the version to ensure consistent behavior

---

## 🔒 About `terraform.lock.hcl`

This file is auto-generated by:

```bash
terraform init
```

It ensures:

* consistent provider versions
* reproducible deployments

It should be committed to version control.

## 🚀 How to Run

### 1. Initialize the project

```bash
terraform init
```

Downloads providers and prepares the environment.

---

### 2. Preview changes

```bash
terraform plan
```

Shows what Terraform will create or modify.

---

### 3. Apply configuration

```bash
terraform apply
```

Creates the resources in Azure.

---

### 4. (Optional) Destroy resources

```bash
terraform destroy
```

Deletes everything created by Terraform.

---

## 📤 Expected Result

After running `terraform apply`, you will have:

* A Resource Group: `my-terraform-rg`
* A Storage Account: `mysac65754273ab`

You can verify them in the **Azure Portal**.

## 🌍 Replication Type

```hcl
account_replication_type = "GRS"
```

**GRS (Geo-Redundant Storage)** means:

* data is replicated locally
* and in a secondary region for disaster recovery

---

## ⚠️ Important Notes

* Storage account names must be **globally unique**
* Costs may apply depending on usage
* This project provisions infrastructure only (no data operations)
* Resource Group deletion behavior is configured to allow cleanup even if it contains resources
